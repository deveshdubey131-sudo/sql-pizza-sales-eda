# Pizza Sales and Order Analysis using SQL

# About :

This project analyzes a pizza sales dataset to uncover insights into customer ordering behavior, revenue generation, product performance, and time-based trends. It uses SQL to perform end-to-end analysis—from basic aggregations to advanced window functions—helping simulate real-world business intelligence scenarios in the food delivery domain.

# Purpose of the Project :

To analyze sales performance and revenue drivers
To understand customer ordering patterns
To identify top-performing products and categories
To support decision-making in pricing, inventory, and marketing
To practice SQL for business-oriented data analysis

# Skills Required :

SQL (Intermediate to Advanced)
Joins (INNER JOIN across multiple tables)
Aggregations (SUM, COUNT, AVG)
Window Functions (RANK, cumulative sums)
CTEs (Common Table Expressions)
Time-based functions (HOUR, date grouping)
Data-driven thinking for business insights

# Different Analysis Done :

1. Basic Sales Analysis
Total number of orders
Total revenue generated
Highest-priced pizza identification
Most common pizza size
2. Product Performance Analysis
Top 5 most ordered pizza types (by quantity)
Top 3 pizzas by revenue
Revenue contribution by each pizza type
3. Category-Level Analysis
Total quantity ordered per category
Category-wise distribution of pizzas
Top-performing pizzas within each category
4. Time-Based Analysis
Order distribution by hour of the day
Average pizzas ordered per day
Daily revenue trends
Cumulative revenue over time
5. Advanced Analytical Insights
Percentage contribution of each pizza type to total revenue
Running (cumulative) revenue analysis
Ranking of pizzas within each category using window functions

# Data Understanding :

Orders Table → order timing (date & time)
Order Details → quantity per order
Pizzas Table → pricing and size
Pizza Types Table → name and category
